# Regras do Projeto (Flighty IAN)

- **Sincronização com o GitHub**: Sempre que houver integração com Git/GitHub e acesso ao repositório remoto, todas as alterações, correções e novos recursos implementados devem ser obrigatoriamente commitados e enviados para o repositório remoto (`git push`) de forma automática ao término de cada tarefa ou ciclo de alteração, garantindo o deploy automático e sincronização imediata sem depender de solicitação manual do usuário.
